## Design Context

### Users
Figurability is for bioinformatics researchers, lab collaborators, and technically fluent readers who need to scan a growing chart library quickly, compare visualization patterns, and open the underlying script or template data without friction. They usually arrive with a practical task: find a reusable figure pattern, inspect how it was built, and decide whether it fits a downstream analysis or publication need.

### Brand Personality
Precise, editorial, trustworthy.

The interface should feel curated rather than promotional, confident rather than flashy, and intellectually clear rather than product-marketing heavy. It should evoke confidence, calm, and a sense that the figure library is maintained with research discipline.

### Aesthetic Direction
Light-first scientific editorial design.

Use a paper-toned base, dark ink-like typography, and a restrained teal-to-cranberry accent system that feels closer to annotated journal spreads than SaaS dashboards. The page should read like a curated atlas of visualization methods, with subtle motion and structured annotation details.

References:
- Editorial layouts with strong hierarchy and generous whitespace
- Scientific posters and journal front matter

Anti-references:
- Generic dark-mode AI landing pages
- Glassmorphism-heavy UI
- Startup-style metric tiles and gradient-driven dashboards

### Design Principles
1. Make the library feel curated: each chart should read like a selected method, not a random file card.
2. Preserve fast scanning: category, preview, code access, and sample data access must stay obvious at a glance.
3. Use restrained distinction: memorable typography and layout should create identity without compromising trust.
4. Let structure carry personality: rhythm, annotation, and composition should do more work than decorative effects.
5. Keep research workflows first: the page should remain lightweight, GitHub Pages-friendly, and easy to extend as new charts are added.
